<Update type="added" label="2025-12-17" description="0.1.0">
**Complete CLI Interface**: Implemented a full command-line interface with 10 subcommands:
- `modix init` - Initialize default configuration
- `modix list` - List all configured models with status indicators
- `modix switch` - Switch between different models
- `modix status` - Show current model status
- `modix add` - Add new model configurations
- `modix remove` - Remove model configurations
- `modix show` - Show vendor details including API keys
- `modix update` - Update existing vendor configurations
- `modix check` - Check configuration for various development tools
- `modix path` - Show configuration file path
</Update>

## Versioning

This project follows [Semantic Versioning](https://semver.org/) guidelines:

- **Major**: Breaking changes that are not backward compatible
- **Minor**: New features that maintain backward compatibility
- **Patch**: Bug fixes and minor improvements

## Changelog Format

This changelog follows [Keep a Changelog](https://keepachangelog.com/) format with sections for:

- **Added**: New features and capabilities
- **Changed**: Modifications to existing functionality
- **Deprecated**: Features that will be removed in future versions
- **Removed**: Features that have been removed
- **Fixed**: Bug fixes and improvements
- **Security**: Security-related changes

## Release Process

1. All changes are tracked in this changelog
2. Version numbers follow semantic versioning
3. Releases are tagged in Git with version numbers
4. Release notes are generated from this changelog

## Contributing to the Changelog

When contributing changes:

1. Add entries to the appropriate section (Added, Changed, Fixed, etc.)
2. Use clear, descriptive language
3. Group related changes together
4. Include links to issues, pull requests, or documentation when relevant
5. Update version number and date when creating a new release

## Backward Compatibility

The project aims to maintain backward compatibility:

- **Configuration Files**: Existing configuration files should work with new versions
- **CLI Interface**: Command-line interface changes will be minimized and documented
- **API Changes**: Breaking changes will be clearly documented in release notes
- **Migration Guides**: Migration guides will be provided when necessary for breaking changes

## Security Notes

Security-related changes are logged in the changelog with appropriate detail while maintaining security best practices.

## Migration Guides

For breaking changes, migration guides will be provided to help users transition to new versions.
