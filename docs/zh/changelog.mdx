<Update type="added" label="2025-12-17" description="0.1.0">
**多供应商 LLM 支持**: 添加了对多个 AI 模型供应商的支持，包括 DeepSeek、Alibaba (Bailian)、ByteDance (VolcEngine) 和 Anthropic
</Update>

<Update type="added" label="2025-12-17" description="0.1.0">
**完整的 CLI 界面**: 实现了完整的命令行界面，包含 10 个子命令：
- `modix init` - 初始化默认配置
- `modix list` - 列出所有配置的模型并显示状态指示器
- `modix switch` - 在不同模型之间切换
- `modix status` - 显示当前模型状态
- `modix add` - 添加新的模型配置
- `modix remove` - 删除模型配置
- `modix show` - 显示供应商详细信息，包括 API 密钥
- `modix update` - 更新现有供应商配置
- `modix check` - 检查各种开发工具的配置
- `modix path` - 显示配置文件路径
</Update>

<Update type="added" label="2025-12-17" description="0.1.0">

## 版本控制

该项目遵循 [语义化版本控制](https://semver.org/) 指南：

- **主版本号**: 不向后兼容的破坏性变更
- **次版本号**: 向后兼容的新功能
- **修订号**: 错误修复和小幅改进

## 变更日志格式

此变更日志遵循 [Keep a Changelog](https://keepachangelog.com/) 格式，包括：

- **新增**: 新功能和功能
- **变更**: 对现有功能的修改
- **废弃**: 将在未来版本中删除的功能
- **移除**: 已删除的功能
- **修复**: 错误修复和改进
- **安全**: 与安全相关的变更

## 发布流程

1. 所有变更都在此变更日志中跟踪
2. 版本号遵循语义化版本控制
3. 发布在 Git 中使用版本号标记
4. 发布说明从此变更日志生成

## 为变更日志做贡献

贡献变更时：

1. 将条目添加到适当的章节（新增、变更、修复等）
2. 使用清晰、描述性的语言
3. 将相关变更分组
4. 在相关时包含指向问题、拉取请求或文档的链接
5. 创建新发布时更新版本号和日期

## 向后兼容性

该项目旨在保持向后兼容性：

- **配置文件**: 现有配置文件应与新版本一起使用
- **CLI 界面**: 命令行界面的变更将最小化并记录
- **API 变更**: 任何破坏性变更将在发布说明中明确记录
- **迁移指南**: 在需要时为破坏性变更提供迁移指南

## 安全说明

安全相关变更在变更日志中记录，详细信息适当，同时保持安全最佳实践。有关安全漏洞报告，请使用 GitHub Issues 跟踪器。
